# PDF 소책자 변환 웹서비스 초간단 기획

## 서비스 정의
사용자가 PDF를 드롭하면 바로 소책자용 PDF로 변환해서 다운로드하게 하는 웹서비스.

## 핵심 원칙
- 한 화면
- 한 기능
- 설명 최소화
- 가입 없음
- 업로드 후 바로 결과 제공

## 사용자 흐름
1. 페이지 접속
2. PDF 파일 드롭
3. 자동 변환 시작
4. 변환 완료
5. 소책자 PDF 다운로드

## MVP 기능
- PDF 드래그앤드롭 업로드
- PDF만 허용
- 페이지 수 확인
- 4의 배수가 아니면 빈 페이지 자동 추가
- 소책자 인쇄 순서로 재배치
- 2-up 형태의 새 PDF 생성
- 다운로드 버튼 제공

## 넣지 않는 것
- 회원가입
- 결제
- 설정 패널
- 미리보기 고도화
- 편집 기능
- 여러 출력 옵션
- 장기 파일 저장

## 화면 구성
### 메인 화면 하나만 사용
- 큰 드롭존
- 짧은 문구: `PDF를 놓으면 소책자 PDF로 바꿉니다`
- 처리 중 상태 텍스트
- 완료 후 다운로드 버튼

## 기본 동작 정책
- 업로드 즉시 변환
- 실패하면 짧은 에러 메시지 표시
- 변환 파일은 일정 시간 뒤 삭제

## 권장 기술 스택
- Next.js
- TypeScript
- Tailwind CSS
- `pdf-lib`

## 처리 방식
- 프론트: 파일 드롭, 상태 표시, 다운로드 처리
- 서버: PDF 읽기, 페이지 순서 계산, 새 PDF 생성

## 핵심 로직
- 총 페이지 수 확인
- 4의 배수가 아니면 빈 페이지 추가
- 소책자 인쇄용 페이지 순서 계산
- 한 장에 두 페이지씩 배치한 새 PDF 생성

## 성공 조건
- 사용자가 설명 없이 바로 쓸 수 있어야 함
- 3단계 이내로 끝나야 함
- 첫 버전은 "정확하게 변환된다" 하나만 만족하면 됨

## 개발 우선순위
1. 드롭존 UI
2. 업로드 API
3. 소책자 변환 로직
4. 다운로드 연결
5. 에러 처리

## 제품 문구
- 메인 문구: `PDF를 소책자 PDF로 바로 변환`
- 보조 문구: `파일을 드롭하면 자동으로 인쇄용 순서로 바꿉니다`
- 버튼 문구: `PDF 드롭` / `다운로드`

## 결론
이 서비스는 도구가 아니라 버튼 하나짜리 유틸리티로 봐야 한다.  
첫 버전은 `PDF 드롭 -> 변환 -> 다운로드` 외의 모든 기능을 버리는 게 맞다.
